Kotlin Flow

Flow 基本用法

注意使用 Flow 的代码与先前示例的下述区别：

  • 名为 flow 的 Flow 类型构建器函数。
  • flow { … } 构建块中的代码可以挂起。
  • 函数 simple 不再标有 suspend 修饰符。
  • 流使用 emit 函数 发射 值。
  • 流使用 collect 函数 收集 值。

冷流：flow{} 中的代码直到被 collect{} 才开始运行，是冷流。而且 collect{} 调用多少次 flow{} 中的代码就运行多少次

Flow 可以被取消，在其父协程被取消后就会被取消。

流构建器

flow {} 是最基础的一个，除此之外还有如下：

  • flowOf 构建器定义了一个发射固定值集的流
  • 使用 .asFlow() 扩展函数，可以将各种集合与序列转换为流。

过渡流操作符

可以使用操作符转换流，就像使用集合与序列一样。 过渡操作符应用于上游流，并返回下游流。 这些操作符也是冷操作符，就像流一样。这类操作符本身不是挂起函数。它运行的速度很快，返回新的转换流的定义。

类似 RxJava 的操作符
虽然操作符本身不是挂起函数，但是传入的参数是有协程上下文的，可以在其中直接调用挂起函数

输出：

list asFlow it = 3
list asFlow it = 4

转换操作符

在流转换操作符中，最通用的一种称为 transform。它可以用来模仿简单的转换，例如 map 与 filter，以及实施更复杂的转换。 使用 transform 操作符，我们可以 发射 任意值任意次。

限长操作符

限长过渡操作符（例如 take）在流触及相应限制的时候会将它的执行取消。

末端流操作符

末端操作符是在流上用于启动流收集的挂起函数。 collect 是最基础的末端操作符。
但是还有另外一些更方便使用的末端操作符：

  • 转化为各种集合，例如 toList 与 toSet。
  • 获取第一个（first）值与确保流发射单个（single）值的操作符。
  • 使用 reduce与 fold 将流规约到单个值。

流上下文

默认，flow { ... } 构建器中的代码运行在相应流的collect {}提供的上下文中。

可以通过，flowOn函数，可以更改流发射的上下文。

流取消检测

为方便起见，流构建器对每个发射值执行附加的 ensureActive 检测以进行取消。 这意味着从 flow { ... } 发出的繁忙循环是可以取消的:

it = 1
it = 2
it = 3
Exception in thread “main” kotlinx.coroutines.JobCancellationException: BlockingCoroutine was cancelled; job=BlockingCoroutine{Cancelled}@2df32bf7

但是，出于性能原因，大多数其他流操作不会自行执行其他取消检测。 例如，如果使用 IntRange.asFlow 扩展来编写相同的繁忙循环， 并且没有在任何地方暂停，那么就没有取消的检测；
